# 고객용 상품목록 화면 기능명세

## 기본 진입
- QR을 통해 특정 매장 / 존 / 구역에 진입한다.
- 최초 진입 시 현재 QR의 구역 상품이 기본 노출된다.

## 검색 기능
- 검색창 제공
- 검색 기준: 제품명 또는 제품코드
- 검색 범위: 현재 매장 전체 상품
- 검색어 입력 시 현재 구역 상품목록 대신 매장 전체 검색 결과 노출
- 검색어 삭제 시 현재 선택 탭의 구역 상품목록으로 복귀

## 구역 탭 기능
- 같은 매장 내 전체 구역 탭 목록 노출
- 기본 선택 탭은 현재 QR 진입 구역
- 탭 클릭 시 해당 구역 상품 목록 노출
- 탭 이동 시 견적은 유지

## 상품 카드 노출 정보
- 상품 이미지
- 상품명
- 사이즈 상세
- 멤버십가

## 상품상세 기능
- 상품 대표 이미지
- 상품명
- 제품코드
- 멤버십가
- 색상 선택
- 사이즈 선택
- 사이즈 없는 상품은 Standard 표시
- 매장 전시 여부
- 수량 선택
- 견적서 담기 버튼
- 제품 상세 링크 버튼

## 견적 정책
- 다른 QR 이동 시에도 견적 유지
- 당일 자정까지 유지
- 직원 전달 전까지 고객 브라우저에 유지

## 디자인 방향
- DESKER 스타일
- 미니멀, 여백 중심, 절제된 UI
- 고객용은 모바일 우선 반응형
## 고객용 UX 수정 방향

### 1. 상품목록 화면 UX
- 상품목록 화면은 세로 리스트형이 아니라 DESKER 레퍼런스와 유사한 이미지 중심 그리드형이어야 한다.
- 상단에는 브랜드 헤더와 검색창이 있어야 한다.
- 상단 우측에는 현재 구역 또는 존을 나타내는 필터가 있어야 한다.
- 예: PANGYO 드롭다운 또는 구역 선택 UI
- 구역 필터를 통해 현재 매장 내 다른 구역 상품으로 전환 가능해야 한다.

### 2. 상품카드 UX
- 상품카드에는 상품 이미지가 중심이 되어야 한다.
- 카드에는 아래 정보가 노출된다.
  - 상품명
  - 사이즈 상세
  - 멤버십가
- 상품카드 전체 클릭 시 상품상세 화면으로 이동해야 한다.
- 현재처럼 카드 우측의 작은 "담기" 버튼 중심 UX가 아니라, 카드 클릭 → 상세 진입 구조가 우선이다.

### 3. 상품상세 화면 UX
- 상품상세 화면은 별도 상세 화면 또는 우측 패널/모달 형태로 열릴 수 있다.
- 화면에는 아래 항목이 포함되어야 한다.
  - 상품 대표 이미지
  - 상품명
  - 제품코드
  - 색상 선택
  - 사이즈 선택
  - 매장 전시 여부
  - 수량 선택
  - 총 상품 금액
  - 견적서 담기 버튼
- 사이즈 없는 상품은 Standard로 표시한다.

### 4. 견적서 UX
- 하단 고정 "견적서 보기" 버튼만 있는 구조가 아니라,
- 담은 상품 목록과 총 금액을 볼 수 있는 견적 요약/견적서 영역이 필요하다.
- 견적서에는 아래가 포함되어야 한다.
  - 담은 상품 목록
  - 옵션 정보(색상 / 사이즈 / 수량)
  - 총 금액
  - 전체 삭제
  - 직원에게 전송 버튼
- 하단 고정형 요약 바 + 펼침형 견적서 구조도 가능하다.

### 5. 구현 우선순위
1. 상품목록 화면 UX를 DESKER 이미지 중심 그리드로 수정
2. 상품 클릭 시 상품상세 화면 구현
3. 견적서 담기 기능 연결
4. 하단 견적서 요약/목록 화면 구현
5. 직원에게 전송 버튼 연결
## 구역 필터 전체 옵션 정책
- ZONE 필터의 최상단에는 "전체" 옵션이 있어야 한다.
- "전체" 선택 시 현재 매장 내 모든 구역에 진열된 상품을 한 번에 조회할 수 있어야 한다.
- 특정 구역 선택 시에는 해당 구역 상품만 조회한다.
- 검색어가 있을 때는 기존 정책대로 현재 매장 전체 상품 검색을 유지한다.
- 검색어가 없고 ZONE이 "전체"이면 현재 매장 전체 진열 상품 목록을 노출한다.
## 반응형 프리뷰 토글 정책
- 고객용 화면은 기본적으로 브라우저 폭에 따라 자동 반응형으로 동작해야 한다.
- 개발/검수 단계에서는 상단에 Desktop / Tablet / Mobile 프리뷰 토글을 둘 수 있다.
- 프리뷰 토글 클릭 시 실제 기기처럼 강제 폭 미리보기를 적용할 수 있어야 한다.
- Desktop / Tablet / Mobile 토글은 검수용 기능이므로 운영 배포 시 노출 여부를 제어할 수 있어야 한다.
## 상품상세 UX 추가 수정

### 1. 상품상세 팝업 위치
- 상품상세 팝업은 현재처럼 화면 하단에 과하게 붙은 형태가 아니라, 화면 중앙에 더 가깝게 보여야 한다.
- 모바일 우선 구조를 유지하되, 바텀시트와 모달의 중간 형태처럼 시각적으로 너무 아래에 치우치지 않게 조정한다.

### 2. 상품 이미지 비중
- 상품상세 팝업 내 상품 이미지는 현재보다 더 크게 노출되어야 한다.
- 사용자가 옵션 선택 전에 제품 이미지를 충분히 인지할 수 있어야 한다.

### 3. 제품 상세보기 버튼
- 상품상세 팝업 내 사이즈 선택 영역 근처에 `제품 상세보기` 버튼이 있어야 한다.
- 버튼 클릭 시 해당 상품에 연결된 상세 URL로 이동해야 한다.
- URL이 없는 경우 버튼은 숨기거나 비활성 처리한다.

### 4. 관리자 입력 항목 추가 필요
- 관리자 페이지 상품 마스터 또는 상품 업로드 항목에 `제품 상세 URL` 필드가 필요하다.
- 이 값은 고객용 상품상세 팝업의 `제품 상세보기` 버튼과 연결된다.
## 상품상세 옵션 연동 정책 추가

### 1. 사이즈별 가격 연동
- 상품상세 화면에서 사이즈 선택 시, 선택된 사이즈에 연결된 상품코드 또는 옵션코드 기준 가격이 표시되어야 한다.
- 즉, 총 금액은 단순 기본가가 아니라 현재 선택된 사이즈 기준 가격으로 계산되어야 한다.
- 수량 변경 시에도 선택된 사이즈 기준 단가 * 수량으로 총 금액이 갱신되어야 한다.

### 2. 색상별 이미지 변경
- 상품상세 화면에서 색상 선택 시 상단 대표 이미지가 선택된 색상에 맞게 변경되어야 한다.
- 이를 위해 상품 데이터에는 색상별 이미지 URL 정보가 필요하다.

### 3. 관리자 관리 항목 추가 필요
- 관리자 상품 마스터 또는 업로드 양식에는 아래 항목이 필요하다.
  - 색상별 이미지 URL
  - 사이즈별 가격 또는 옵션별 가격
  - 제품 상세 URL

### 4. 현재 구현 단계 정책
- 이번 단계에서는 mock data 기준으로
  - 색상 선택 시 이미지 변경
  - 사이즈 선택 시 가격 변경
  - 총 금액 즉시 갱신
  을 우선 구현한다.
  ## 매장 전시 여부 판단 정책 상세화

### 1. 전시 여부 판단 기준
- 상품상세 화면의 전시 여부는 현재 선택된 사이즈 또는 옵션에 연결된 제품코드 기준으로 판단한다.
- 즉, 전시 여부는 상품 단위가 아니라 선택된 옵션(사이즈) 기준이다.

### 2. 전시 여부 판정 방식
- 현재 매장에서 ZONE 필터를 `전체`로 두었을 때 조회되는 제품코드이면 `전시 중`으로 표시한다.
- 현재 매장에서 ZONE 필터를 `전체`로 두었을 때 조회되지 않는 제품코드이면 `전시 없음`으로 표시한다.

### 3. 상세 화면 연동 규칙
- 사용자가 사이즈를 변경하면 전시 여부도 함께 즉시 갱신되어야 한다.
- 즉, 사이즈 선택 시
  - 가격 변경
  - 총 금액 변경
  - 전시 여부 변경
  가 동시에 반영되어야 한다.

### 4. 현재 구현 단계
- 이번 단계에서는 mock data 기준으로
  - 사이즈별 제품코드
  - 현재 매장 전체 전시 제품코드 목록
  을 비교해 `전시 중 / 전시 없음`을 표시한다.
  ## QR 생성 및 연결 정책

### 1. QR 생성 단위
- QR코드는 각 존 또는 세부 구역별로 고유하게 생성할 수 있어야 한다.
- 하나의 존이 여러 세부 구역으로 나뉘는 경우, 각 세부 구역별로 각각 다른 QR을 생성할 수 있어야 한다.

### 2. QR 진입 동작
- 고객이 QR코드를 스캔하면, 해당 QR에 연결된 매장 / 존 / 구역의 상품 목록이 기본 화면으로 노출되어야 한다.
- 기본 선택 구역은 QR에 연결된 현재 구역이어야 한다.

### 3. QR 관리 기능
- 메인관리자는 QR 생성, 다운로드, 인쇄, 사용/미사용 전환을 할 수 있어야 한다.
- QR은 매장, 존, 구역 정보와 연결되어야 한다.

### 4. QR 데이터 구조
- 각 QR은 아래 정보를 가진다.
  - 매장 ID
  - 존 ID
  - 구역 ID
  - QR명
  - 연결 URL
  - 사용 여부
  ## 상품 마스터 엑셀 업로드 정책

### 1. 업로드 권한
- 상품 기본정보의 엑셀 일괄 업로드는 마스터 관리자만 가능하다.
- 매장 관리자는 상품 기본정보를 수정할 수 없다.

### 2. 업로드 대상 필드
- 제품명
- 제품코드
- 색상코드
- 제품 이미지 URL
- 멤버십 가격
- 제품 상세 페이지 URL

### 3. 업로드 방식
- 마스터 관리자는 엑셀 양식을 다운로드할 수 있어야 한다.
- 양식에 맞는 엑셀 파일 업로드 시 상품 기본정보를 일괄 등록/수정할 수 있어야 한다.

### 4. 반영 정책
- 동일한 기준키가 존재하면 수정
- 존재하지 않으면 신규 등록
- 삭제는 즉시 삭제보다 사용중지 또는 비노출 처리 방식을 우선 검토한다.
